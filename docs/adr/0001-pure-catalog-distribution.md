# 纯目录分发：npm 包零组件内容

安装器 npm 包只包含 CLI 代码与目录元数据（catalog），不携带任何
组件内容。所有组件——包括维护者自己编写的 skills——一律以官方
上游链接收录，按官方安装方式安装；自有组件须先发布为独立公开
仓库才能进目录。

**Status**: accepted

**Considered Options**: 混合分发（自有组件打包进 npm、第三方走
官方渠道，即上一轮会话的"Vendored Payload"方案，未落盘）。被
否决：安装器仓库需要维护组件内容副本，与上游产生同步漂移；纯
目录模式把内容维护成本归零，安装器仓库只对元数据负责。

**Consequences**: 自有组件在发布为独立仓库之前无法进入目录；
目录按维护者节奏渐进补充官方链接。上一轮"分发简化版 CCG 引擎
（保留 8 子系统）"的设想随之作废——任何第三方工具（含 CCG）只能
以官方原样安装的组件条目收录，不存在改造后再分发的形态。
