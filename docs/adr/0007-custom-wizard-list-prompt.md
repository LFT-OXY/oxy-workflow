# 向导自写列表组件（回车返回 + Esc 后退）

向导链的「返回上一步」原是 `@inquirer/checkbox` 列表里的一个
复选项：回车在多选里永远是"提交整屏"，返回必须先空格勾选再
回车，用户实测反馈该交互反直觉。原生组件无法区分"回车时光标
停在哪一行"，也不暴露 Esc 按键钩子，此需求没有配置层面的解法。

决定：基于 `@inquirer/core` 自写一个列表组件（多选 / 单选两种
模式），向导三类屏（宿主屏、类型屏、汇总确认屏）全部换用。
按键语义统一为：

- 光标停在「← 返回」行按回车 = 直接后退一层；
- 光标在普通行时，空格勾选（多选模式）、回车提交进入下一步；
- 屏幕任意位置按 Esc = 后退一层（宿主屏后退即回主菜单，确认屏
  后退即回最后一个类型屏）；
- 每屏保留可见的「← 返回」行作为入口，Esc 只是快捷方式；按键
  提示固定渲染在列表底部。

随组件一并确立间距规范：向导屏在问题行与选项之间、返回行与
条目列表之间、列表与底部按键提示之间各空一行，返回行下以细
分隔线与条目区隔；主菜单每个分组标题前空一行。

代价与边界：自行维护约百行的按键与渲染逻辑，放弃原生组件的
免维护性；manage 流程与主菜单本次不换，仍用原生 select（其
"返回"是单选项，回车即生效，不存在同样的问题）。

**Status**: accepted
