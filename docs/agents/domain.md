# 领域文档

工程 skill 探索代码库时如何消费本仓库的领域文档。
本仓库为**单上下文**布局。

## 探索前先读

- 根目录 **`CONTEXT.md`** —— 项目领域术语表
- **`docs/adr/`** —— 阅读与当前工作区域相关的 ADR

若这些文件不存在，**静默继续**。不要提示缺失，也不要建议提前创建
——`/domain-modeling` skill（经 `/grill-with-docs` 与
`/improve-codebase-architecture` 触达）会在术语或决策真正敲定时
惰性创建它们。

## 文件结构

```
/
├── CONTEXT.md
├── docs/adr/
│   ├── 0001-<决策>.md
│   └── 0002-<决策>.md
└── src/
```

## 使用术语表的词汇

输出中出现领域概念时（issue 标题、重构提案、假设、测试名），使用
`CONTEXT.md` 定义的术语，不要漂移到术语表明确避免的同义词。

所需概念尚不在术语表时，这是信号——要么你在发明项目不用的语言
（重新考虑），要么存在真实缺口（记下来交给 `/domain-modeling`）。

## 标记 ADR 冲突

输出与现有 ADR 相矛盾时，显式指出而非静默覆盖：

> _与 ADR-0007（xxx）相矛盾——但值得重开，因为……_
